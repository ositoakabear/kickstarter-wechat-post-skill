# 写作风格与结构模板

通用原则：不要原样照抄源文案；按平台风格精简改写。两个平台内容保持事实一致，**改一处同步另一处**。

## 小红书 markdown

风格：第一人称、口语化、轻 emoji、短句、有「被种草」感。控制在手机一屏多滑几下能读完。

结构模板：

```markdown
# 本周好物推荐：<钩子标题，制造好奇/反差>

<开场一句话点出产品 + 最大亮点/获奖背书>

(配图 1: <主图说明，如 实物挂墙图>)

🐦 **<亮点小标题>：**
<2-4 行解释，口语化>

(配图 2: <动态/GIF 说明>)

✨ **<对比/状态小标题>：**
- **【常态】** ...
- **【死态】** ...

(配图 3: <成对对比图说明>)

✨ **核心升级：**
1. **<点>**：...
2. **<点>**：...

(配图 4: <App/细节图说明>)

📊 **货比三家：**
- **<本品>**：... | ... | ...
- **<竞品A>**：...
- **<竞品B>**：...

🐻 **"<栏目名>"观察员<昵称>点评：**
> "<一句有观点的总结>"

⚠️ **众筹小贴士**：Kickstarter 项目存在交付延期与售后不确定性，理性支持。

#垂直标签1 #垂直标签2 #品类 #人群 #本周好物推荐
```

要点：
- 用 `(配图 N: 说明)` 占位，正文里讲清每张图位置与内容。
- 成对状态（常态/死态、前/后）要写明左右或先后，避免后续图摆反。
- 标签：3-9 个，含品类 + 人群 + 固定栏目标签。

## 公众号 HTML

风格：比小红书更完整、信息更全，但仍要精简有节奏。分小节，配图穿插。排版用内联样式（微信不支持外部 CSS / class 不稳）。**渲染坑详见 wechat-pitfalls.md**。

建议结构（每节一个 `<section>`）：
1. 开场 + 获奖/数据背书（封面图在草稿封面，不在正文顶部重复）
2. 创意/故事由来
3. 基本信息（项目名、众筹时间/金额、核心功能）
4. 成对状态对比图（用 `<table>` 并排，文字在图下）
5. Pro 升级点（自定义 `●` span 列表）
6. App/细节展示图
7. **竞品对比：竖向卡片**，每个维度一张卡，本品高亮 `🏆`
8. 痛点/风险（客观）
9. 目标受众与市场
10. 「观察员点评」引用块（署名按用户约定）
11. 官方资源导航（链接）+ 众筹风险提示

标题：统一前缀便于系列化，如 `本周好物推荐：<标题>`。

正文配图：先在 HTML 里用本地路径或外链，脚本会自动上传到微信 CDN 并替换（见 SKILL.md 第 6 步）。
